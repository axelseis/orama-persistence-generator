API Reference

This document provides a complete reference for our REST API.

Authentication

All API requests require authentication using API keys.

curl -H "Authorization: Bearer YOUR_API_KEY" https://api.example.com/v1/users

Endpoints

Our API provides the following main endpoints:

Users

Manage user accounts and profiles.

  • GET /users - List all users
  • POST /users - Create a new user
  • GET /users/{id} - Get user details

Projects

Manage projects and their settings.

  • GET /projects - List all projects
  • POST /projects - Create a new project
  • PUT /projects/{id} - Update project

Error Handling

The API returns standard HTTP status codes and error messages.

{
  "error": "Invalid API key",
  "code": 401
}
