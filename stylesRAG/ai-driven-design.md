# AI-Driven Design / Extreme Personalization

## Definition
A dynamic, data-informed style where UI adapts to user context, behavior, and preferences. Layouts, content, and emphasis can shift in real time to maximize relevance.

## Intent and use
- Deliver personalized experiences at scale.
- Ideal for content platforms, onboarding flows, and adaptive products.

## Color palettes (examples)
- **Adaptive Base**: #F6F7FB (light), #DDE3EA (gray), #9AA7B3 (steel), #2E3740 (dark), #111418 (text)
- **Soft Tech**: #F7F8FA (light), #E2E7EE (cool gray), #B2C0CC (mid), #4B5A66 (dark), #1B1F24 (text)
- **Dynamic UI**: #FFFFFF (white), #EEF1F5 (light), #C9D3DD (mid), #2F3A45 (dark), #0F141A (text)

## Design rules
- **Modularity**: UI built from flexible blocks that can reorder.
- **Personalization**: adjust hierarchy, density, and content based on user signals.
- **Clarity**: personalization should never hide core actions.
- **Feedback**: explain why changes happen when possible.

## Typography (fonts available in Penpot)
Use stable, legible sans for dynamic layouts.
- **Headings**: "Inter", "IBM Plex Sans", "Plus Jakarta Sans", "Manrope", "Sora"
- **Body**: same family in regular/medium
- **UI**: same family in medium/semibold

## Layout and components
- **Cards** and **modules** that can reorder without breaking.
- **Personalized panels** with clear boundaries.
- **Recommendations** styled as distinct, scannable blocks.

## Illustration and imagery
- Minimal; prioritize clarity and quick comprehension.
- Use subtle animations to guide attention, not distract.

## Accessibility
- Avoid over-personalization that causes confusion.
- Keep navigation consistent and discoverable.

## Agent keywords
personalization, adaptive, dynamic, modular, data-driven, contextual, user-centric
